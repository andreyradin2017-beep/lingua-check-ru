## 🛠 Оптимизация токенов (rtk)

Для экономии 60-90% токенов и увеличения длины сессий (проект "ЭкономимТокенов"):

- **Обязательно** используй `rtk` в качестве префикса для часто используемых CLI-утилит:
  `git`, `npm`, `ls`, `grep`, `find`, `cat`, `curl`, `pip`, `python`, `pytest`.
- **Пример**: `rtk git status` вместо `git status`.
- Если `rtk` недоступен или команда возвращает ошибку, используй оригинальную команду без прокси (fallback).

## 📜 Глобальные правила разработки

### 1. Использование MCP (Model Context Protocol)

Всегда используй подключенные MCP-инструменты для получения актуальных данных:

| Инструмент | Назначение |
|------------|------------|
| **Context7 / Google Developer Knowledge** | Документация по библиотекам, API, примеры кода |
| **Google Search** | Свежая информация, которой нет в базе знаний |
| **Supabase** | Анализ структуры БД, запросы, управление данными |
| **Render** | Управление деплоем, мониторинг сервисов |
| **Sequential Thinking** | Сложные логические задачи, планирование |

### 2. Коммуникация

- Ответы и планы реализации пиши только на **русском языке**.
- Не стесняйся задавать уточняющие вопросы.

### 3. Технологический стек

| Компонент | Технологии |
|-----------|------------|
| **Frontend** | React 19, TypeScript 5.9, Mantine UI 8, Vite 8 (Rolldown) |
| **Backend** | FastAPI 0.135, SQLAlchemy 2.0 (Async), Pydantic v2, Playwright |
| **База данных** | PostgreSQL (Supabase), SQLAlchemy 2.0.48, asyncpg 0.31, redis 7.4 |
| **Анализ** | pymorphy3 (морфология) |
| **Тесты** | Vitest 4 (frontend), pytest 8.3 (backend), Playwright (E2E) |
| **Bundler** | Rolldown (Rust) — 10-30x быстрее Rollup |
| **Plugin** | @vitejs/plugin-react v6 (Oxc вместо Babel) |
| **Linter** | ESLint 10, typescript-eslint 8.57 |

**Важно:**
- По умолчанию используй **Node.js и TypeScript**.
- Не добавляй TS в зависимости — проект использует нативную поддержку Node.js.
- Используй **ESLint** для проверки кода.

### 4. Качество кода и логирование

- Обязательно добавляй логи в релевантных местах, используй адекватный уровень логирования.
- После правок актуализируй тесты и документацию.
- После завершения задачи всегда запускай линтеры и автотесты:

```bash
# Frontend
npm run lint
npm run test

# Backend
cd backend && python -m pytest tests/ -v --cov=app
```

## 🏗 Архитектура проекта

### Структура директорий

```
russian-lang/
├── backend/
│   ├── app/
│   │   ├── routers/          # API endpoints (scans, texts, trademarks, exceptions)
│   │   ├── services/         # Бизнес-логика (scan_service, token_service)
│   │   └── models.py         # SQLAlchemy ORM модели
│   ├── tests/                # Backend тесты (90%+ покрытие)
│   ├── manage.py             # CLI утилиты (init, seed, update-counts)
│   └── run.py                # Точка входа
├── src/
│   ├── pages/                # Страницы приложения (ScanPage, HistoryPage, TextPage)
│   ├── components/           # Переиспользуемые компоненты
│   ├── utils/                # Утилиты (url.ts, sanitize.ts, translations.ts)
│   └── config/               # Конфигурация (api.ts)
├── tests/                    # E2E тесты (Playwright)
├── docs/                     # Документация
├── scripts/                  # Вспомогательные скрипты
├── full_diagnostic.py        # Диагностика системы
└── monitor_services.py       # Мониторинг сервисов
```

### Ключевые сервисы

| Сервис | Файл | Назначение |
|--------|------|------------|
| **Scan Service** | `backend/app/services/scan_service.py` | Сканирование сайтов, параллелизм (5 воркеров) |
| **Token Service** | `backend/app/services/token_service.py` | Анализ текста, in-memory кэширование словарей |
| **Dictionary Cache** | In-memory | Кэш словарных слов (ускорение в 10 раз) |
