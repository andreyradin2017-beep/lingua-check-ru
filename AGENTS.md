# Правила проекта LinguaCheck-RU (AGENTS.md)

Этот файл содержит глобальные правила для ИИ-агентов и инструкции по оптимизации работы.

## 🛠 Оптимизация токенов (rtk)

Для экономии 60-90% токенов и увеличения длины сессий (проект "ЭкономимТокенов"):

- **Обязательно** используй `rtk` в качестве префикса для часто используемых CLI-утилит:
  `git`, `npm`, `ls`, `grep`, `find`, `cat`, `curl`, `pip`, `python`, `pytest`.
- **Пример**: `rtk git status` вместо `git status`.
- Если `rtk` недоступен или команда возвращает ошибку, используй оригинальную команду без прокси (fallback).

## 📜 Глобальные правила разработки

1. **Использование MCP (Model Context Protocol)**: Всегда используй подключенные MCP-инструменты для получения актуальных данных:
   - **Context7 / Google Developer Knowledge**: Для документации по библиотекам, API и примеров кода.
   - **Google Search**: Для поиска свежей информации, которой нет в базе знаний модели.
   - **Supabase**: Для анализа структуры БД, выполнения запросов и управления данными.
   - **Render**: Для управления деплоем и мониторинга сервисов.
   - **Sequential Thinking**: Для решения сложных логических задач и планирования.
2. **Коммуникация**:
   - Ответы и планы реализации пиши только на **русском языке**.
   - Не стесняйся задавать уточняющие вопросы.
3. **Технологический стек**:
   - По умолчанию используй **Node.js и TypeScript**.
   - Не добавляй TS в зависимости, если нет явной просьбы — используй нативную поддержку Node.js.
   - Используй **ESLint** для проверки кода.
4. **Качество кода и логирование**:
   - Обязательно добавляй логи в релевантных местах, используй адекватный уровень логирования.
   - После правок актуализируй тесты и документацию.
   - После завершения задачи всегда запускай линтеры и автотесты.

## 💡 Рекомендации

- При работе с большими файлами старайся читать только необходимые фрагменты.
- Используй `rtk grep` или `rtk find` для быстрого поиска по проекту.
