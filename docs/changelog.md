## [1.6.0] - 9 марта 2026

### 🔴 Удалено

- **Тип нарушения «Визуальное доминирование»** (`visual_dominance`) из frontend
  - Причина: Сложность реализации, низкая востребованность
  - Backend логика сохранена в `backend/app/services/visual_dominance_service.py`
  
- **Колонка «Вес (%)»** из таблицы нарушений на странице `/scans`
  - Удалены поля `visual_weight_foreign` и `visual_weight_rus`
  
- **Кнопка «Все в бренды»** (массовое добавление нарушений в бренды)
  - Причина: Риск случайного добавления лишних слов
  
- **Модальное окно «Бренды»**
  - Удалена отдельная страница управления брендами
  - Функционал доступен через контекстные кнопки

### ✨ Добавлено

- **Кнопка «Пометить как исключение»** на странице `/scans`
  - Иконка: `IconShieldCheck` (зеленый цвет)
  - Функция: `addException()` — добавление в `global_exceptions`
  - Мгновенное уведомление об успехе
  
- **Tooltip для всех кнопок действий** (Mantine UI)
  - `Tooltip label="Посмотреть скриншот" withArrow`
  - `Tooltip label="Пометить как бренд" withArrow`
  - `Tooltip label="Добавить в глобальные исключения" withArrow`
  - `Tooltip label="Перейти на страницу" withArrow`
  
- **Tooltip для полей формы сканирования**
  - URL сайта: «Введите адрес сайта для проверки на соответствие ФЗ №168-ФЗ»
  - Глубина: «Количество уровней ссылок для проверки (1-5)»
  - Скриншоты: «Создаёт скриншоты нарушений, что может увеличить время сканирования»
  
- **Счетчик символов** в Textarea на странице `/text`
  - Формат: `{text.length} симв.`
  - Расположение: `rightSection`
  
- **Пагинация** на странице `/history`
  - Порог: 20 записей на страницу
  - Компонент: `Pagination` (Mantine)
  - Автосброс при удалении записей

### 🐛 Исправлено

- **Горизонтальный скролл таблицы** на мобильных устройствах
  - Файл: `src/pages/ScanPage.tsx`
  - Решение: Обертка `<div style={{ overflowX: 'auto' }}>`
  
- **Недостаточный размер touch-целей** (<44px)
  - Файл: `src/pages/HistoryPage.tsx`
  - Решение: `ActionIcon size="lg"` (44px автоматически)
  
- **Контраст dimmed текста** (WCAG AA)
  - Файл: `src/index.css`
  - Изменение: `--mantine-color-dimmed: #5c5f66` (вместо `#868e96`)
  - Контраст: 3.9:1 → 4.5:1
  
- **Пагинация в HistoryPage** (зависание при 0 записей)
  - Файл: `src/pages/HistoryPage.tsx`
  - Решение: Условие `scans.length > ITEMS_PER_PAGE && activePage > 0`
  - Дополнительно: `useEffect` для сброса страницы при изменении количества записей
  
- **Backend: NameError при импорте exceptions router**
  - Файл: `backend/app/main.py`
  - Решение: Добавлен импорт `from app.routers import ..., exceptions`
  
- **Frontend: Vite слушал только IPv6**
  - Файл: `vite.config.ts`
  - Решение: `server: { host: '0.0.0.0', port: 5173 }`
  - Результат: Доступен и по `localhost`, и по `127.0.0.1`

### ♿ Доступность

- **ARIA-labels** для всех интерактивных иконок
  ```tsx
  <ActionIcon aria-label="Удалить запись" size="lg">
    <IconTrash size={16} aria-hidden="true" />
  </ActionIcon>
  ```
  
- **Навигация:**
  ```tsx
  <NavLink aria-label={`Перейти на страницу ${item.label}`} />
  ```
  
- **Reduced-motion поддержка**
  ```css
  @media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
      animation-duration: 0.01ms !important;
      transition-duration: 0.01ms !important;
    }
  }
  ```

### 📱 Адаптивность

- **Адаптивный padding** на странице 404
  - Было: `py={120}`
  - Стало: `py={{ base: 60, md: 120 }}`
  
- **Адаптивный padding** на главной странице
  - Было: `p={50}`
  - Стало: `p={{ base: 24, sm: 50 }}`

## [1.5.3] - 8 марта 2026

### Оптимизация

- Удалено визуальное доминирование (избыточные нарушения)
- Polling оптимизирован: 3с (сканирование), 15с (история)
- Увеличен чанк вставки нарушений: 100 → 200
- Добавлен debounce для поиска: 300мс
- Ограничение отображения нарушений: max 500

### Исправления

- Ошибки TypeScript в `HistoryPage.tsx`
- Совместимость с Mantine 8
- Исправлен горизонтальный скролл на мобильных (A1)
