## 3. Тест-кейсы

### TC-001: Лемматизация

**Цель:** Проверить корректность лемматизации.

**Входные данные:**
```
батарея, батарею, батареи, батарее, батареей, батарее
```

**Ожидаемый результат:**
Все формы маппятся на лемму `батарея`.

---

### TC-002: Визуальное доминирование (Backend only)

**Цель:** Проверить правило visual_weight.

**Входные данные:**
- Foreign: `COFFEE TO GO` (font-size: 40px, font-weight: 700)
- Russian: `Кофе с собой` (font-size: 12px, font-weight: 400)

**Ожидаемый результат:**
```
weight_foreign = 0.6*40 + 0.3*700 + 0.1*contrast = 24 + 210 + ... = 234+
weight_rus = 0.6*12 + 0.3*400 + 0.1*contrast = 7.2 + 120 + ... = 127+

234 > 1.5 * 127 = 190.5 → Нарушение visual_dominance
```

**Примечание:** Функционал удален из frontend, доступен только в backend.

---

### TC-003: Сканирование с таймаутом

**Цель:** Проверить обработку таймаутов.

**Входные данные:**
- URL: `http://timeout.example.com` (не отвечает >120с)

**Ожидаемый результат:**
- Статус страницы: `timeout`
- Сканирование продолжается для других страниц
- Общий статус: `completed` или `stopped`

---

### TC-004: Исключения (Trademarks)

**Цель:** Проверить работу исключений.

**Шаги:**
1. Добавить бренд: `CoffeeMaster`
2. Запустить проверку текста с `CoffeeMaster`
3. Проверить отчет

**Ожидаемый результат:**
- Слово помечается как `trademark`
- Не считается нарушением

---

### TC-005: Глобальные исключения

**Цель:** Проверить работу global_exceptions.

**Шаги:**
1. Добавить исключение: `gmp`
2. Запустить сканирование с словом `gmp`
3. Проверить отчет

**Ожидаемый результат:**
- Слово не помечается как нарушение
- Исключение применяется глобально

---

### TC-006: Экспорт отчетов

**Цель:** Проверить экспорт в различные форматы.

**Шаги:**
1. Запустить сканирование
2. Дождаться завершения
3. Экспортировать в Excel
4. Экспортировать в PDF
5. Экспортировать в CSV

**Ожидаемый результат:**
- XLSX: Корректная таблица с колонками
- PDF: Читаемый отчет с форматированием
- CSV: UTF-8 кодировка, кириллица корректна
