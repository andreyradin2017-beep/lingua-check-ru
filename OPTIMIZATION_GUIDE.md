# 🚀 Руководство по оптимизации LinguaCheck-RU

## 📊 Проблема больших сканов

При сканировании крупных сайтов (100+ страниц) возникали проблемы:

1. **1000+ нарушений** в таблице
2. **Время загрузки >30 секунд**
3. **Падение браузера** от нагрузки
4. **Нечитаемые данные**

## ✅ Решение

### 1. Группировка нарушений (Backend)

**Endpoint:** `GET /api/v1/scan/{id}/grouped`

**Что делает:**
- Объединяет одинаковые нарушения на странице
- Считает количество повторений (count)
- Сохраняет до 3 контекстов
- Сортирует по частоте (убывание)

**Пример:**

```json
// Было (1000 записей)
[
  {"word": "нутриентах", "type": "unrecognized_word", "page_id": "..."},
  {"word": "нутриентах", "type": "unrecognized_word", "page_id": "..."},
  ...
]

// Стало (94 записи)
[
  {
    "word": "нутриентах",
    "type": "unrecognized_word",
    "count": 80,
    "page_url": "https://example.com/page",
    "text_context": "...",
    "contexts": ["...", "...", "..."]
  }
]
```

### 2. Лимит на выдачу (Backend)

**Параметр:** `GET /api/v1/scan/{id}?limit=1000`

**Зачем:**
- Защита frontend от перегрузки
- Контроль размера ответа
- Быстрая загрузка

### 3. Оптимизированный Frontend

**Изменения в ScanPage.tsx:**

```typescript
// Загрузка сгруппированных данных
const checkStatus = async (id: string) => {
  const res = await axios.get(`/api/v1/scan/${id}`);
  setResult(res.data);
  
  // Загружаем группировку если есть нарушения
  if (res.data.summary.total_violations > 0) {
    const groupedRes = await axios.get(`/api/v1/scan/${id}/grouped`);
    setGroupedViolations(groupedRes.data);
  }
};

// Отображение с группировкой
{groupedViolations.map((v) => (
  <Table.Tr key={v.id}>
    <Table.Td>{v.word}</Table.Td>
    <Table.Td>
      <Badge color={v.count > 50 ? 'red' : v.count > 10 ? 'orange' : 'blue'}>
        x{v.count}
      </Badge>
    </Table.Td>
    ...
  </Table.Tr>
))}
```

## 🎯 Как использовать

### Backend разработчику

```bash
# Получить сгруппированные нарушения
curl http://127.0.0.1:8000/api/v1/scan/{id}/grouped

# Получить с лимитом
curl http://127.0.0.1:8000/api/v1/scan/{id}?limit=500

# Получить все (не рекомендуется)
curl http://127.0.0.1:8000/api/v1/scan/{id}?limit=10000
```

### Frontend разработчику

```typescript
// Всегда используй группировку для больших сканов
const { data: grouped } = useQuery(
  ['scan', scanId, 'grouped'],
  () => axios.get(`/api/v1/scan/${scanId}/grouped`)
);

// Отображай count в таблице
<Badge color={count > 50 ? 'red' : 'blue'}>
  x{count}
</Badge>
```

### Пользователю

1. Открой сканирование
2. Если нарушений много — увидишь **зеленый alert**
3. Таблица показывает **"слово xN"** вместо N одинаковых строк
4. **Цвет бейджа** показывает частоту:
   - 🔴 Красный: >50 раз
   - 🟠 Оранжевый: 10-50 раз
   - 🔵 Синий: <10 раз

## ⚠️ Важные замечания

1. **Группировка работает для каждой страницы отдельно**
   - "слово" x5 на странице A
   - "слово" x3 на странице B
   - Это 2 разные записи

2. **Сохраняются до 3 контекстов**
   - Для отладки и понимания
   - Не перегружает таблицу

3. **ID берется от первого нарушения**
   - Для кнопок действий (в бренды/исключения)
   - Все одинаковые слова удалятся разом
