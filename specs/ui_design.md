# UI Design Spec: LinguaCheck‑RU (Mantine UI)

## 1. Общие принципы

- UI‑библиотека: **Mantine UI (React)**.
- Тема: светлая по умолчанию.
- Шрифт: Inter (или стандартный шрифт Mantine, с возможностью легко сменить на Inter).
- Цвета:
  - primary: синий (Mantine theme `blue`)
  - secondary: зелёный (`green`)
  - danger: красный (`red`)
  - warning: оранжевый (`orange`)
  - success: светло‑зелёный

Интерфейс только на русском языке.

---

## 2. Страницы

### 2.1. `/` (Главная)

- `AppShell`/`Container`.
- Заголовок (`Title`): «Проверка сайтов и текстов на соответствие требованиям по русскому языку».
- Два `Card`:
  1. Карточка "Проверить сайт":
     - `TextInput` для URL,
     - `Button` "Начать сканирование".
  2. Карточка "Проверить текст":
     - `Textarea` для текста,
     - кнопка "Загрузить файл" (TXT/DOCX/PDF),
     - `Button` "Начать проверку текста".

### 2.2. `/scans`

- `Table` со списком сканов:
  - дата,
  - URL,
  - статус,
  - уровень риска (цветной `Badge`).
- При клике на строку — переход на `/scans/:id`.

### 2.3. `/scans/:id`

- `Tabs`:
  - "Карта сайта" — список страниц с индикатором риска.
  - "Все нарушения" — список нарушений (по типам).
  - "Статистика" — общие числа (страниц, нарушений, проценты).

Элементы:

- для нарушения — `Card` с:
  - типом (`Badge`: foreign_word / visual_dominance),
  - контекстом текста,
  - если visual_dominance: показывать значения visual_weight_foreign / visual_weight_rus.

### 2.4. `/texts/:id`

- Оригинальный текст (в `ScrollArea`).
- Таблица/список нарушений с контекстом.
- Кнопки "Экспорт CSV" / "Экспорт PDF".

---

## 3. Ограничения

- Не писать кастомный CSS, если можно решить через Mantine‑props.
- Не плодить уникальные стили кнопок — использовать один стиль кнопки из Mantine.

---

## 4. Запланированные улучшения UX/UI (По итогам аудита)

1. **История (/history)**:
   - Автоматический поллинг статусов каждые 10 секунд (переход «В процессе» -> «Завершено»).
   - Консистентность кнопок: кнопки действий в таблице (Отчет, Удалить) должны выглядеть единообразно (например, обе как `ActionIcon` с тултипами или обе с текстом).
   - Информативное «Empty State» (заглушка с иконкой).

2. **Сканирование (/scans)**:
   - Добавление Toast-уведомления при успешном запуске сканирования и предложение перейти в Историю.
   - Базовая UI-валидация поля URL.
   - Замена комментариев типа "( Phase 5 )" на понятные текстовые описания, например "Расширенный анализ доменных зон".

3. **Анализ текста (/text)**:
   - Исправление отключенной кнопки при программной вставке текста.
   - Сохранение истории локальных проверок в `localStorage` (до 5-10 штук).
