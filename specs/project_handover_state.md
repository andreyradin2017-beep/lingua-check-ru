# Состояние проекта LinguaCheck-RU (Handover Document)

Этот лог-файл содержит актуальное состояние проекта для быстрого погружения ИИ-агента в новом контексте.

## Архитектура и Стек Технологий

**Backend:**

- Python 3.12 + FastAPI
- SQLAlchemy (СУБД SQLite для MVP)
- Pymorphy3 (Лемматизация и морфологический анализ слов)
- Playwright (Краулинг сайтов, извлечение текста и CSS)
- Pydantic (Валидация данных, схемы API)

**Frontend:**

- Node 22 + React + Vite + TypeScript
- Mantine UI + Tabler Icons
- Axios (HTTP-клиент)
- Papaparse (экспорт CSV), jsPDF (экспорт PDF), SheetJS (XLSX)

## Что уже реализовано (Фазы 1-16)

1.  **Парсинг и Нормализация**:
    - Извлечение текста со страниц, исключение email, url, имен собственных, сокращений, одиночных кириллических букв.
    - Использование `pymorphy3` для поиска нормальной формы (`normal_form`) и сверка со словарями.
2.  **Классификация нарушений**:
    - `foreign_word` (иностранная лексика), `typo` (опечатка), `not_translated` (без перевода), `visual_dominance` (доминирование иностранного над русским по CSS), `possible_trademark` (возможно товарный знак), `trademark` (товарный знак).
3.  **UI/UX (ScanPage)**:
    - Индикаторы сканирования, статистика страниц и нарушений.
    - Группировка нарушений по страницам (Accordion), вывод слова и контекста (snippet).
    - Мультиселекторы для фильтрации типов нарушений.
    - Пагинация, поиск, экспорт в Excel/PDF.
4.  **Управление Брендами (Фазы 15-16)**:
    - CRUD API для товарных знаков (`/api/v1/trademarks`).
    - Модальное окно управления брендами прямо на странице результатов (`ScanPage`).
    - Динамическая фильтрация: добавленные бренды на лету исключаются из списка обычных нарушений во фронтенде.
    - Массовое действие "Все отфильтрованные в бренды".
5.  **История сканирований**:
    - Таблица "История" (`HistoryPage.tsx`) со списком прошлых сканирований. Загрузка старых отчетов по ID.

## Текущие проблемы и Задачи (Фаза 17)

_(То, на чем мы остановились и что нужно сделать следующим шагом)_

В проекте накопился небольшой технический долг, который был описан в плане **Фазы 17**:

1.  **Frontend (TypeScript)**: В файлах `TextPage.tsx` и `HistoryPage.tsx` все еще присутствуют переменные типа `any` (12 ошибок `eslint`). Нужно навести порядок в типах (`TextCheckResult`, `TextViolation`).
2.  **Backend (Тесты)**: Отсутствует покрытие `pytest` для новых роутеров `/api/v1/scans` и `/api/v1/trademarks`.
3.  **Frontend (UX)**: В модальном окне брендов (`ScanPage.tsx`) удаление бренда по клику на корзину происходит мгновенно. Необходимо обернуть эту функцию в диалог подтверждения `modals.openConfirmModal` ("Вы уверены, что хотите удалить бренд?").

## Как запускать

- **Backend:** `cd backend` -> `.\.venv\Scripts\activate` -> `uvicorn app.main:app --reload` (порт 8000)
- **Frontend:** `cd frontend` -> `npm run dev` (порт 5173)
