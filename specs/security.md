# Security & Validation Spec

## 1. Валидация ввода

- URL:
  - должен начинаться с `http://` или `https://`.
  - запрещать `javascript:` и `data:` схемы.
- Ограничения:
  - `max_depth` и `max_pages` ограничить (например, max_depth ≤ 5, max_pages ≤ 1000).
- Файлы:
  - разрешённые форматы: TXT, DOCX, PDF.
  - максимальный размер файла: 10 МБ.

## 2. Безопасность API

- Разрешать запросы только с фронтенд‑происхождения (CORS).
- Предусмотреть базовую аутентификацию/ключ доступа, если в будущем потребуется.

## 3. Ограничения сканирования

- Сканировать только домен, явно введённый пользователем.
- Не сохранять чувствительные личные данные, найденные в тексте страниц.
